# 09 — SOP

Standard operating procedures: the "Systems Over Heroics" value (PEGS-000
Art. III §5) made real. If it happened twice, it becomes an SOP; once an SOP
is stable, it becomes an automation candidate (`13-Automation`).

**Convention**

- Filename: `SOP-<company-or-ENT>-<topic>.md` (ENT = enterprise-wide)
- Every SOP is written from [12-Templates/SOP-TEMPLATE.md](../12-Templates/SOP-TEMPLATE.md)
- Every SOP names one owner, one trigger, and its automation status

**Delegation by design**

SOPs are Class 3 territory: they are ratified by the accountable company
lead or role holder — **not** the Founder — unless they touch a firewall
(clinical/legal/PHI, brand, compliance), which escalates them per PEGS-000
Art. VI §4.

**Status:** Active — populated in Phase 4.
**Custodian:** each SOP's named owner.
