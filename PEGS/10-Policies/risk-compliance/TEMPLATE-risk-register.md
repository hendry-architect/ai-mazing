# Risk Register — <Enterprise / Entity> — <YYYY-QN>

> Named risks with named owners. Reviewed quarterly with the plan; a risk
> nobody owns is a surprise on a schedule.

| ID | Risk | Category | Likelihood (1-5) | Impact (1-5) | Score | Owner | Mitigation (funded?) | Early-warning signal | Review |
|---|---|---|---|---|---|---|---|---|---|
| R-01 | Founder key-person concentration | People | | | | founder | Succession roadmap + deputies + insurance | Deputy paths untested >6mo | Q |
| R-02 | Provider departure (clinic capacity) | People | | | | | Pipeline + retention + coverage plan | Schedule utilization spikes | Q |
| R-03 | PHI breach / cyber incident | Firewall | | | | | Cyber policy + training + IR SOP | Phishing reports, audit gaps | Q |
| R-04 | Payer/regulatory change hits core service | Regulatory | | | | | Matrix watch + service diversification | Rule proposals in pipeline | Q |
| R-05 | Cash concentration / runway | Financial | | | | | Treasury ladder + 13-wk forecast | Days-cash nears floor | M |
| R-06 | Brand/reputation event | Brand | | | | | Brand canon discipline + IR comms annex | Review-rating drift, viral risk | Q |
| R-07 | Key vendor/platform dependency | Operational | | | | | Exit plans + data portability | Contract/renewal cliffs | Q |

**Scoring:** score = likelihood × impact; ≥15 = red (mitigation must be
funded in the current quarter's plan); 8–14 amber; <8 watch.

**Lifecycle:** new risks enter with owner + signal defined · retired risks
move to a RETIRED section with date and reason · every red risk appears in
the quarterly plan's risk section by rule.
