# Decision Log — <body / entity> — <YYYY>

> The standing, searchable record of decisions for one body or entity —
> rolled up from Meeting Kit minutes (D-IDs), resolutions, consents, and
> Class 2 memos. One row per decision; the log points to records, it never
> replaces them. Feeds the enterprise decision index ⚙
> (`14-Knowledge-Base/KNOWLEDGE-SYSTEM.md` §4).

| D-ID / Ref | Date | Decision (one line) | Class | Decided by | Advice/dissent noted | Record (link) | Status | Confidence |
|---|---|---|---|---|---|---|---|---|
| D-2026-001 | 2026-07-20 | <e.g., Mission approved to Phase B> | 2 | Chairman | none | <instrument link> | in force | High |
| | | | | | | | in force / superseded / reversed | High / Medium / Low |

## Rules

- Logged within 48h of the decision (usually by the follow-up-report step ⚙).
- **Advisory decisions:** where a council advised, note the range of advice
  and whether the decision followed it — that trail teaches future stewards
  (council charter, decision-log rule).
- Reversals are new rows referencing the old (the log never erases —
  PEGS-000 Art. VIII §3.4).
- Quarterly: the body's chair scans the log — decisions made but not
  executed become action items, not amnesia.
