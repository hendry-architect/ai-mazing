# 14 — Knowledge Base

The institutional memory layer: ratified knowledge that is true but not
binding — reference material, not rules. "Teach what you master" (PEGS-000
Art. VII) lives here.

**Belongs here**

- Glossary of Enterprise terms (defined terms from the canon in one place)
- Ratified session learnings and postmortems — lessons enter here only by
  explicit ratification, never by drift (Art. VI §2)
- Onboarding reading paths: what a new executive, advisor, or family member
  reads, in order (PEGS-000 first, always)
- FAQ maintained from real questions

**Distinction that matters:** if it binds behavior, it is a policy (`10`) or
SOP (`09`), not knowledge. This folder explains; it does not command.

**Status:** Active — populated continuously.
**Custodian:** Founder until a knowledge role is chartered.
