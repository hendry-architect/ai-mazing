# Travel & Executive Retreat System

Retreats and strategic travel are governance instruments, not perks: the
Enterprise thinks in decades in rooms it chose deliberately. Every retreat
runs the Meeting Kit chain; every trip has a playbook, a budget, and a
report — or it's vacation (also honorable, just not this system).

**Library contents**

| Artifact | Use |
|---|---|
| [TEMPLATE-quarterly-retreat.md](TEMPLATE-quarterly-retreat.md) | Quarterly leadership retreat agenda + report shell |
| [PLAYBOOK-travel.md](PLAYBOOK-travel.md) | Domestic playbook + international strategy summit protocol |
| [TEMPLATE-trip-budget-itinerary.md](TEMPLATE-trip-budget-itinerary.md) | One file: budget + itinerary + logistics |

**The rhythm**

| Event | Cadence | Anchors |
|---|---|---|
| Quarterly leadership retreat (half/full day, local) | Q1–Q3 | Quarter score + next-quarter plan lock |
| Annual retreat (1–2 days, off-site) | Q4 | Council Q4 + constitutional review + annual plan (framework in `04-Advisory-Council`) |
| International strategy summit | As ratified (Class 2 if > budget threshold) | One decade-question, deep work |

**Meeting Kit rule:** agenda before departure · decisions and action items
captured in the room · retreat report within 72h · archives with the
quarter. A retreat without a report was a nice trip.

**Automation hooks:** retreat dates pre-calendared annually ⚙ · budget
rows → finance sheet ⚙ · itinerary → calendar ICS + travel folder ⚙ ·
action items → tracker like any meeting ⚙.
