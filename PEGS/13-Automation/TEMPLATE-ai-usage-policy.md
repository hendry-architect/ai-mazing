# AI Usage Policy — <Enterprise>

Implements the Automation Doctrine (PEGS-000 Art. VI §5): automate the
repeatable, humanize the exceptional, hard stop at the firewalls.

## 1. Permitted by default

Drafting, research, summarization, coding, content production, analysis,
translation review, workflow automation — inside the human-review rules
below and the brand canon of the entity doing the work.

## 2. Human review requirements (non-negotiable)

| Output class | Review before it acts/ships |
|---|---|
| Clinical content, medical claims | Licensed physician — always (firewall) |
| Legal commitments, contracts | Founder + counsel |
| Anything public under a brand | Brand owner per its canon (Founder for SOSTENGO) |
| Payments, financial transactions | Human per Authority Matrix — AI never initiates money movement |
| Governance documents | The ratifying authority (merge = ratification) |
| Internal drafts, research, code | Owner's judgment; tests/verification for code |

**Principle:** AI output is a draft until a named human owns it. Every
automated system has a human owner, an inspection method, and a firewall
boundary (Automation Catalog).

## 3. Data rules

- No PHI into any AI tool without a BAA in force and Founder approval —
  named tool, named data, named safeguard.
- `06-Trust` contents and unratified legal/deal matters: Founder-approved
  tools only.
- Prefer enterprise/zero-retention tiers; personal accounts never carry
  Enterprise data.

## 4. Disclosure & integrity

We do not present AI-fabricated experiences as lived ones (Integrity before
advantage). Claims cited to real sources; the Founder's voice is never
synthesized for public use without his explicit per-instance approval.

## 5. Violations

Firewall breaches follow incident response (`10-Policies/risk-compliance/`)
— surface early, never punished for reporting; hiding one always is.
