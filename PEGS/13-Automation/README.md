# 13 — Automation

The leverage layer: where the Automation Doctrine (PEGS-000 Art. VI §5) is
implemented. Repeatable work is documented (`09-SOP`), then templated
(`12-Templates`), then automated here — in that order, and never past a
firewall.

**Belongs here**

- The automation register: every automated system, its named human owner, its
  inspection method, and its firewall boundary
- Automation build specs and runbooks

**Backlog** (sequenced after Phase 2 ratification)

1. **Registry lint (CI):** every PEGS doc has a valid control block,
   references PEGS-000, and appears in the master registry
2. **Scaffolding script:** new doc from template with ID and control block
   pre-filled
3. **Annual review trigger:** scheduled reminder + checklist for the
   constitutional review (Art. VI §7)
4. **Canon publishing:** render `main` into a private, readable canon site
   for executives, advisors, and family

**Rules of this folder**

- An automation without a named human owner does not run (Art. VI §5).
- Clinical, legal, and PHI workflows are never automated past human
  accountability (Art. VI §4.1).

**Status:** Active — register created with the first automation.
**Custodian:** Founder until a technology role is chartered.
