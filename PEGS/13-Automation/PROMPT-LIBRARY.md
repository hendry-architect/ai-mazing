# Prompt Library

Ratified, reusable prompts — the Enterprise's operating commands. Session
improvisations graduate here only by explicit ratification (PR), same as
any canon (Art. VI §2).

## Registered command sets

| Command | Domain | Canonical source |
|---|---|---|
| `/SOSTENGO` | Founder personal brand deliverables | Command Registry v2.0 (skill: command-registry) |
| `/PH` | PassQual Health growth system | Command Registry v2.0 |
| `/PHSEO` | passqual.com + GBP SEO | Command Registry v2.0 |

The Command Registry remains the single canonical source for those three;
this library indexes it and holds everything else.

## Governance prompts (use verbatim, fill <>)

**New governance doc:** "Act as CoS/Architect of Systems for PEGS. Draft
<doc> in <folder> from 12-Templates/PEGS-DOC-TEMPLATE.md, anchored to
PEGS-000, as DRAFT v0.1.0 on a branch with a draft PR. Do not self-ratify."

**Decision memo:** "Prepare a Class 2 decision memo for <decision> using
12-Templates/DECISION-MEMO-TEMPLATE.md. Steelman option C (do nothing) and
run the Five Gates honestly — a failed gate is a finding, not a problem to
wordsmith away."

**Meeting run:** "Generate the Meeting Kit chain for <meeting> on <date>:
agenda from the tracker + <topics>; packet checklist; after minutes are
pasted, produce follow-up report and updated action tracker."

**Quarter close:** "Compile 11-Meetings/records/<YYYY>/Q<N>: quarter index
(meetings, D-IDs, action closure rate), flag overdue actions by owner."

**Canon Q&A:** "Answer from the PEGS canon only, citing document and
section. If the canon is silent, say so and propose where the answer should
live."

## Contribution rule

New prompt → used successfully ≥2 times → PR into this file with an
`id`-style name and its use-case. Prompts touching firewalled domains note
their human-review requirement inline.
