# PEGS Executive Experience (PEX™)

> The Founder-facing layer of PEGS. Eight short documents that let the
> Chairman operate the entire enterprise **through Claude, in plain
> language, without ever opening GitHub**. PEX creates no governance —
> it translates the governance that already exists (per Chairman's
> Directive No. 001 §VII: refinement over expansion). These eight files
> are the upload set for the Founder's Claude Project.

## Rules of this layer

1. **Written for one reader** — the Founder, in second person, plain
   language, no governance jargon. Anyone else reads it over his
   shoulder.
2. **Projection, not source** — PEX describes canon; it never defines
   it. If PEX and a canon artifact disagree, canon wins and PEX gets
   corrected (design principle 7).
3. **Zero GitHub assumption** — every flow in these pages is operable
   entirely from Claude Project and Claude Chat.
4. **Maintained on ratification** — when canon changes in a way the
   Founder would feel (a cadence, a class rule, a rhythm), the CGO
   updates the affected PEX page in the same sync pass (W-7).
5. **The cheat sheet stays one page.** Forever. If it can't fit, the
   system got too complicated — fix the system, not the page.

Custodian: Founder. Operator: CGO function.
